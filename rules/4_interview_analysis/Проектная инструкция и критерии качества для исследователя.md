## Антипаттерны для исключения

❌ **НЕ упрощать:** "[GENERIC_ROLE]" вместо конкретных функций работы
❌ **НЕ обобщать:** "[GENERIC_ACTIVITY]" вместо конкретных [SPECIFIC_ACTIVITIES]
❌ **НЕ терять связи:** между характеристиками [PRODUCT_TYPE] и ожидаемыми результатами
❌ **НЕ обрабатывать несколько респондентов одновременно** при консолидации
❌ **НЕ смешивать ожидания и решения:** функциональные ожидания должны отвечать на "ЗАЧЕМ?", решения на "ЧТО ДОЛЖНО БЫТЬ?"
❌ **НЕ использовать абстрактные формулировки:** избегать "[ABSTRACT_TERMS]" без конкретики
❌ **НЕ терять детали:** сохранять имена, цифры, конкретные примеры, [CONTEXT_DETAILS]

## Критерии качества итогового результата

- **Полнота:** 80%+ полей заполнено конкретными данными
- **Специфичность:** 70%+ формулировок содержат конкретные детали
- **Подтверждаемость:** 70%+ утверждений подкреплены цитатами
- **Actionability:** Формулировки [MAIN_PATTERNS] позволяют принимать продуктные решения
- **Консистентность:** Четкое разделение [MAIN_PATTERNS] и решений
- **Детализация:** Максимальная конкретизация контекстов, [PROCESS_TYPES], технологий
- **Структурированность:** Логическая архитектура ожиданий-проблем-решений-оценок

## Плейсхолдеры для заполнения

При использовании данного шаблона для конкретного проекта необходимо заменить следующие плейсхолдеры:

### Общие
- `[PROJECT_NAME]` - название проекта
- `[TARGET_SEGMENT]` - целевой сегмент
- `[PRODUCT_TYPE]` - тип продукта/сервиса
- `[PROFESSIONAL_ROLE]` - профессиональная роль респондентов
- `[PLATFORM_NAME]` - название платформы/сервиса

### Контекстные
- `[CONTEXT_DETAILS]` - детали контекстов к сохранению
- `[DETAILS_EXAMPLES]` - конкретные примеры контекстных деталей
- `[DOMAIN_SPECIFIC_DETAILS]` - специфичные для домена детали
- `[SOLUTION_TYPE]` - тип решения
- `[TECHNICAL_SPECS]` - технические характеристики

### Паттерны
- `[MAIN_PATTERNS]` - ключевые паттерны которые определяют неудовлетворенность пользователей (например, ожидаемые результаты)
- `[RELATED_PATTERNS]` - связанные с ними сопутствующие детали из framework_structure.md (например, проблемы, важность, частотность, удовлетворенность - другие элементы определяющие общий паттерн пользователя)

### Процессные
- `[FEATURE_CHAIN]` - цепочка фич и процессов
- `[SELECTION_CRITERIA]` - критерии выбора
- `[INVESTMENT_TYPE]` - тип инвестиций

### Медиа и коммуникации
- `[VIDEO_LINK]` - ссылки на видео-инструкции
